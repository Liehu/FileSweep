# AppDir 软件目录识别与整理设计文档

## 1. 问题定义

绿色/便携软件的特点是以**目录**为单位存在，目录下包含主程序、DLL、资源文件等。若以单文件粒度扫描，会产生大量噪音记录，破坏去重和分类逻辑。

**核心矛盾**：扫描器当前以单个文件为粒度建立索引，但绿色软件应以目录为单位作为一个逻辑整体处理。

---

## 2. 识别规则

### 2.1 特征检测（只检查目录的直接子文件，不递归）

| 规则编号 | 条件 | 置信度 | 规则标识 |
| :--- | :--- | :--- | :--- |
| R1 | 根目录有 `.exe` + 有 ≥3 个 `.dll` | 0.90 | `exe+dll` |
| R2 | 根目录有 `.exe` + 有文档特征文件 | 0.80 | `exe+doc` |
| R3 | 根目录有且仅有 1 个 `.exe` + 有 1~2 个 `.dll` | 0.70 | `single-exe+dll` |

### 2.2 文档特征文件列表

```
readme.txt / readme.md / readme
license.txt / license.md / licence.txt
release.txt / release_notes.txt / changelog.txt / changes.txt
说明.txt / 使用说明.txt / 使用说明.md / 说明书.txt
帮助.txt / 帮助文档.txt / 版本说明.txt / 更新日志.txt
readme_zh.txt / readme_cn.txt
```

### 2.3 排除的 exe 文件（不认为是主程序）

```
unins* / uninst* / uninstall*
helper* / updater* / update*
crashreport* / crash_report*
setup* / install*
registrator* / register*
elevate* / launcher_helper*
```

### 2.4 主程序推断（pickMainExe）

从非噪音 exe 候选列表中，选取与**目录名归一化后 Levenshtein 距离最小**的 exe 作为主程序代表。

## 3. 数据模型变更

### 3.1 FileRecord 新增字段

```go
// 追加到现有 FileRecord 结构体末尾
IsAppDir     bool   `json:"isAppDir"`     // true = 软件目录记录
AppDirPath   string `json:"appDirPath"`   // 软件目录完整路径
AppDirReason string `json:"appDirReason"` // 识别规则：exe+dll / exe+doc / single-exe+dll
```

### 3.2 软件目录记录的字段语义重定义

| 字段 | 普通文件 | 软件目录 |
| :--- | :--- | :--- |
| `Name` | 文件名 | 目录名去版本号后的软件名 |
| `LocalPath` | 文件完整路径 | 主 exe 的完整路径 |
| `AppDirPath` | 空 | 目录完整路径（整理操作的目标） |
| `FileHash` | 文件 SHA-256 | `SHA-256(目录路径 + 直接子exe列表)` |
| `FileSize` | 文件大小 | 目录递归总大小 |
| `Version` | 从文件名提取 | 从目录名提取 |
| `Extension` | 实际扩展名 | 固定为 `.exe` |

### 3.3 数据库迁移

在 `migrations.go` 的 patches 列表追加三项：

```go
{"file_records", "is_app_dir",     "BOOLEAN DEFAULT FALSE"},
{"file_records", "app_dir_path",   "TEXT DEFAULT ''"},
{"file_records", "app_dir_reason", "TEXT DEFAULT ''"},
```

在 `BatchInsertFileRecords` 和 `InsertFileRecord` 的 SQL 语句中增加这三个字段的读写。

## 4. 扫描器改动（scanner.go）

### 4.1 Scanner 结构体新增字段

```go
type Scanner struct {
    Workers       int
    ProgressCh    chan ScanProgress
    DetectAppDirs bool // 默认 true
}
```

### 4.2 walkDir 逻辑变更

在进入每个**非根子目录**时，先调用 `DetectAppDir(path)`：

```
目录回调中：
  if DetectAppDirs && path != scanRoot:
    sig = DetectAppDir(path)
    if sig.IsAppDir:
      构建 scanEntry{isAppDir:true, appDirPath:path, appDirSig:sig}
      将该目录路径加入 appDirPaths 集合
      return filepath.SkipDir   ← 跳过目录内所有子文件

文件回调中：
  if parentDir in appDirPaths:
    return nil  ← 跳过该文件（已被目录记录覆盖）
```

### 4.3 hashFiles 的 AppDir 分支

```
if e.isAppDir:
    hash = computeDirHash(appDirPath)
        // SHA-256(目录路径 + "|" + 所有直接子exe文件名拼接)
    size = computeDirSize(appDirPath)
        // filepath.WalkDir 递归求和
    version = extractVersionFromDirName(dirBase)
    record.IsAppDir     = true
    record.AppDirPath   = appDirPath
    record.AppDirReason = sig.Reason
    record.LocalPath    = mainExePath
    record.Name         = sig.AppName
else:
    正常计算文件哈希（现有逻辑不变）
```

### 4.4 进度上报格式

软件目录在 walking 阶段上报：
```json
{ "currentFile": "[软件目录] Nmap", "stage": "walking" }
```
在 hashing 阶段上报：
```json
{ "currentFile": "[目录] Nmap", "stage": "hashing" }
```

## 5. AppDir 检测函数（新文件 appdir.go）

```go
// internal/core/appdir.go

type AppDirSignature struct {
    IsAppDir   bool
    MainExe    string  // 推断的主程序文件名
    AppName    string  // 推断的软件名（去版本号）
    Confidence float64
    Reason     string  // exe+dll / exe+doc / single-exe+dll
}

func DetectAppDir(dirPath string) AppDirSignature
func inferAppName(dirPath string) string  // 目录名去版本号
func pickMainExe(candidates, allExes []string, dirName string) string
func isNoiseExe(nameLower string) bool
func isDocFile(nameLower string) bool
```

`inferAppName` 逻辑：调用 `ExtractVersion(dirBase)`，若提取成功则从目录名中去除版本号及尾部 `-_ v` 等分隔符，剩余部分为软件名；提取失败则直接用目录名。

## 6. 执行器改动（executor.go）

### 6.1 新增 moveDir 方法

```go
func (e *Executor) moveDir(srcDir, destDir string) error
```

逻辑：
1. `os.MkdirAll(filepath.Dir(destDir), 0755)`
2. `os.Rename(srcDir, destDir)` — 同盘优先
3. 跨盘降级为 `copyDirRecursive(srcDir, destDir)` + `os.RemoveAll(srcDir)`

### 6.2 Execute 中的分支判断

```go
case OpMove:
    if action.File.IsAppDir && action.File.AppDirPath != "" {
        // 移动整个目录，Dest 为目标目录路径（不含软件目录名本身）
        destDir := filepath.Join(action.Dest, filepath.Base(action.File.AppDirPath))
        err = e.moveDir(action.File.AppDirPath, destDir)
    } else {
        err = e.moveFile(action.Source, action.Dest)
    }
    opLog.CanRevert = true

case OpDelete:
    if action.File.IsAppDir && action.File.AppDirPath != "" {
        if e.UseRecycleBin {
            err = e.recycleFile(action.File.AppDirPath) // 整目录移入回收站
        } else {
            err = os.RemoveAll(action.File.AppDirPath)
        }
    } else {
        // 原有逻辑
    }
```

### 6.3 回退支持

`RevertMove` 已有的逻辑对目录同样适用（`os.Rename` 即可），无需单独处理。

## 7. 分类器适配（classifier.go）

软件目录的 `Extension` 固定为 `.exe`，因此现有分类规则天然将其归入**安装包**类。

若希望软件目录有独立分类，在 `rules.yaml` 中增加：

```yaml
- name: 绿色软件
  target_path: PortableApps
  extensions: []           # 不依赖扩展名
  name_keywords: []        # 不依赖关键词
  app_dir_only: true       # 新增标志：仅匹配 IsAppDir=true 的记录
```

分类器中增加判断：

```go
if rule.AppDirOnly && !file.IsAppDir {
    continue
}
if file.IsAppDir && rule.AppDirOnly {
    return ClassifyResult{Category: rule.Name, TargetDir: rule.TargetPath}
}
```

## 8. 去重器适配（dedup.go）

软件目录的哈希基于目录路径+子exe列表，**不参与基于文件哈希的精确去重**（R1层），只参与版本组去重（R3层）：

```go
// 在 exactHashMatch 开头跳过 AppDir 记录
for i, r := range records {
    if used[i] || r.IsAppDir {
        continue
    }
    byHash[r.FileHash] = append(byHash[r.FileHash], i)
}
```

版本组去重（`versionGroupMatch`）正常处理 AppDir，用 `AppDirPath` 的目录名做 `extractBaseName`，可识别同一软件不同版本的目录：

```
Nmap 7.94/  ->  base="nmap", ver="7.94"
Nmap 7.95/  ->  base="nmap", ver="7.95"
→ 保留 7.95，标记 7.94 为待删除
```
